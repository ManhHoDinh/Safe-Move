## Section 4: How It Works

### Headline

**"From raw data to safe routes in under 90 seconds."**

### Four-Step Flow

Displayed as a horizontal timeline on desktop (connected nodes with lines between them) and vertical stack on mobile. Each step has a large circled number, a bold label, and a description.

---

**Step 1: DATA INGESTION**

Traffic cameras, weather APIs, historical flood records, and crawled news articles feed into FloodRoute AI 24/7. The system ingests structured sensor data alongside unstructured text and imagery. There is no "off" state. The moment conditions change, the system knows.

*Visual: Multiple data source icons flowing into a central node. Icons for camera, cloud/weather, newspaper, database.*

---

**Step 2: AI ANALYSIS**

The LLM evaluates severity from every available signal. Autonomous agents cross-reference camera imagery against weather forecasts against historical patterns against news reports. Each flood point receives a confidence score. Conflicting signals trigger escalation, not suppression.

*Visual: Central processing node with branching analysis paths. Confidence percentage badge animating from 0% to 94%.*

---

**Step 3: SMART ROUTING**

Google Maps integration reroutes traffic in real time. Individual drivers receive push alerts before they reach flooded zones. Fleet operators see updated optimal routes across their entire network. City traffic management systems receive signal timing and lane closure recommendations. All of this happens before the flood becomes visible to the driver.

*Visual: Map node showing a route recalculating -- gray line fading, green line drawing in, with a "23 min saved" badge.*

---

**Step 4: CONTINUOUS LEARNING**

Every flood event, every user report, every prediction outcome, every route deviation feeds back into the models. Reinforcement learning agents evaluate what worked and what didn't. The prediction window expands. The severity accuracy increases. The rerouting gets faster. The system after a monsoon season is not the system before it.

*Visual: Circular feedback node with metrics showing improvement -- "Prediction accuracy: 89% -> 94%" and "Rerouting speed: 4.2s -> 1.1s".*

## Technical Notes for Implementation

### Performance Requirements
- Lighthouse Performance score target: 95+
- First Contentful Paint: < 1.2s
- Largest Contentful Paint: < 2.0s
- Total page weight: < 800KB (excluding demo map tiles)
- All images in WebP/AVIF with responsive srcset
- Critical CSS inlined, remainder loaded async

### Animation Implementation
- Use Framer Motion for scroll-triggered animations (fade-in-up, 400ms ease-out)
- Map demo: Pre-rendered animation sequence using Lottie or canvas, not live Mapbox (for performance)
- Stat counters: Framer Motion spring animation on viewport entry
- Parallax: CSS transform only, respect `prefers-reduced-motion`
- Route recalculation animation: SVG `stroke-dashoffset` animation

### SEO Structure
- H1: Hero headline (single)
- H2: Each section headline
- H3: Sub-elements within sections
- Schema markup: SoftwareApplication, Organization, FAQPage (if FAQ added)
- Meta description: "FloodRoute AI detects urban flooding in real time and reroutes traffic automatically. AI-powered flood prediction, severity assessment, and smart routing for cities and enterprises."
- Open Graph image: Dark map with flood zones and routing visualization

- CTA click events: `hero_cta_click`, `demo_cta_click`, `footer_cta_click`
- Scroll depth: 25%, 50%, 75%, 100%
- Demo interaction: `demo_play`, `demo_replay`, `demo_complete`
